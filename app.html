Lesson 1: The Basics of the Web and HTML

How the Web Works

The web consits out of computers that communicate with each other. When a person goes to a web page, the computer sends a HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen. Generally, the main type of document on the web is HTML.

HTML

HTML stands for Hypertext Markup Language which contains text content which describes "what you see" and markup which describes "how it looks".

The HTML markup consist of an opening tag and a closing tag while the whole thing is called an element, e.g.:

Inline
<b>= Bold
<em>= Italics
<a>= Anchor (add hyperlinks, needs attribute like href)
<img>= Image (no closing tag, needs source and alt text)
<br>= Line Break (no closing tag)
<q>= Short Quotation
<strong>= Defines Important Text

Block
<p>= Paragraph (can give heigth and width)
<body>= Body
<form>= User Input Field

Generally an HTML tag is always contained within angled brackets. Most tags have an opening tag and a closing tag. Some tags (called "void" tags) do not require a closing tag.

Tags and Elements

HTML is written in sentences or elements composed of tags and text. Tags are the markers that tell the text or source how to act or look. In general tags can have multipe attributes, e.g. anchors tags can include links (the tags use the href attribute which includes the actual link).

The image tag for instance uses the "src" attribute. Important here is that the image tag is a so called "void tag". It doesn't have any content and therefore doesn't have a closing tag. An image is included by: "img src="url" alt="text". The text gets discplyed when the images is not there.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them. Text is displayed on multiple lines.

In order to break text one need to use the "br" tag which is an inline-element. Alternatively one can use the "p" tag which stands for paragraph and which is a block-element.

Lesson 2: Creating a Structured Document with HTML

HTML and CSS

Everything on a webpage is a box while the style can be applied through CSS.

HTML and CSS are two programming "languages". While HTML controls the "structure" of a web page, CSS controls the "style" of a page, i.e. how it looks. When programmers talk about the "DOM" (Document Object Model) they are talking about the "tree-like structure" of a page. The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother, and so on...

Lesson 3: Adding CSS Style to HTML Structure

What is CSS?

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class-name and then specifying the style that should apply to that class. Read more re: Cascading Style or CSS tricks.

Avoiding Errors

When programmers don't avoid repetition, they will often have to do the same thing over and over. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere the repeated code appears, problems will arise.

The Box Model

There are four main points to note about box sizing:

1. HTML elements are boxes and each box has 4 components: Margin, border, padding and the content. Please note that the margin is not including in the size of the actual box.
2. Because there are so many components to each box, it can often be hard to get the size of a box just right.
3. There are two techniques one can use to help deal with sizing issues. Set sizes in terms of percents rather than pixels. Set the box-sizing attribute to border-box for every element.
4. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.

Box Positioning
Divs are block elements (as opposed to inline), so by default they take up the entire width of a page. Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
